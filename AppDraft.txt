● Project Description: Telegram Bet Slip Bot

  Overview

  A Telegram bot for a private betting group that monitors for bet slip screenshots submitted via /bet [amount] command,
   uses Claude Vision API to parse bet slip contents (matches, selections, odds), validates the data, and logs
  everything to Google Sheets. Photos are preserved on Google Drive. An admin approval flow ensures only validated bets
  reach the main ledger, with automatic balance tracking per user.

  Core Flow

  User sends photo(s) with /bet [amount] [currency?] caption
    → Bot downloads photos → saves to Google Drive (daily folders)
    → Bot sends photos to Claude Vision API
    → Vision AI extracts: source, bet type, matches/events, selections, odds
    → Bot validates odds where possible (leg multiplication check ±0.02)
    → Bot checks for duplicates (same user, same legs+odds within 24h)
    → Valid → PENDING sheet, bot confirms "✅ Bilet inregistrat"
    → Invalid → FLAGGED sheet silently (no group message)

  Admin reviews PENDING sheet, deletes invalid rows
  Admin runs /approve
    → Moves remaining PENDING rows → MAIN sheet
    → Deducts stakes from BALANCE sheet

  Vision AI Parsing

  The bot sends all photos to Claude Vision API with a structured prompt. It must handle:

  - Bet slip summaries — standard bookmaker view with all legs, odds, total
  - Market selection screens — user screenshotted odds from the market menu before placing
  - Bet history views — from "My Bets" section, different layout
  - Bet builders / same-game multis — multiple selections within one match
  - Social media screenshots — Instagram posts, Twitter/X posts, tournament odds graphics. User might screenshot an
  esports tournament bracket or odds post and bet on a specific outcome (e.g., "10 lei pe NAVI")
  - Informal/non-bookmaker sources — any image that contains identifiable odds or events
  - Any bookmaker — Betano, Superbet, bet365, Unibet, etc.
  - Any sport or esport — football, tennis, CS2, League of Legends, Valorant, etc.
  - Any language on the slip — Romanian, English, mixed
  - Live bets — allowed, no time validation
  - Multi-photo slips — all images sent to Vision API in one request

  For non-standard sources (Instagram posts, tournament graphics), the Vision AI extracts whatever event/selection info
  is visible from the image, and the user's /bet message provides additional context (e.g., /bet 10 pe NAVI → text after
   the amount is the selection).

  Validation Rules

  - Odds check: Where per-leg odds are visible, multiply and compare to total (±0.02 tolerance). Mismatch → FLAGGED
  silently.
  - Duplicate check: Same user, same legs + same odds within 24h → FLAGGED silently.
  - Incomplete slip: Can't extract meaningful event/selection info → FLAGGED silently.
  - All flagging is silent. No message in group chat. Admin reviews FLAGGED sheet independently.

  Google Sheets Structure

  One Google Sheets document with 4 sheets:

  PENDING Sheet (bot writes here)

  ┌──────┬────────┬──────┬───────┬──────┬──────┐
  │ DATA │ PARIOR │ Meci │ PARIU │ COTA │ MIZA │
  ├──────┼────────┼──────┼───────┼──────┼──────┤
  └──────┴────────┴──────┴───────┴──────┴──────┘

  MAIN Sheet (approved bets)

  Same 6 columns + STATUS (manual) + CASTIG (formula). Bot does not touch STATUS or CASTIG.

  FLAGGED Sheet

  Same 6 columns + MOTIV (reason for flagging).

  BALANCE Sheet

  Pure numbers only. One column per user. Each row is a signed number — no descriptions, no text.

  User A | User B | User C
    -50  |        |
   +100  |   -25  |
    -30  |        |   -10

  Admin's formulas calculate running balances.

  Column Details

  Single bet:
  01.03.2026 | Georo | Dinamo - FC Arges | Karamoko 3+ pe poarta, 1 castiga | 4.10 | 50.00 RON

  Multi-leg bet (60-space separator within Meci and PARIU cells for text wrap):
  01.03.2026 | Foitos | UTA - FCSB[60 spaces]Arsenal - Chelsea[60 spaces]Dinamo - FC Arges | FCSB peste 5.5...[60
  spaces]Peste 7.5...[60 spaces]Peste 6.5... | 29.29 | 20.00 RON

  Social media / informal bet:
  20.03.2026 | Alex | IEM Katowice 2026 | NAVI to win | 3.50 | 10.00 RON

  - Meci and PARIU use 60-space separator between entries for multi-leg bets
  - COTA is always total/combined odds. For informal bets where odds aren't visible, left blank for admin to fill
  - MIZA includes currency. Default RON if user only specifies a number. Other currencies stored as-is, no conversion

  Admin identified by Telegram user ID. No other commands exposed to users.

  Bot Chat Behavior

  - Valid bet parsed: "✅ Bilet inregistrat"
  - Flagged bet: Nothing. Silent write to FLAGGED sheet.
  - No photo with /bet: "Trimite biletul (foto) cu /bet [suma]"
  - Buffer timeout: "Adauga /bet [suma] ca sa inregistrez biletul"
  - All other messages: Bot stays completely silent.

  Photo Storage

  - Bot downloads every submitted photo via Telegram API
  - Uploads to Google Drive in daily folders (e.g., 2026-03-20/)
  - Naming: {username}_{timestamp}.jpg
  - Survives Telegram message deletion

  Tech Stack

  - Language: Python 3.11+
  - Telegram: python-telegram-bot library
  - Vision AI: Anthropic SDK — Claude Sonnet for image parsing
  - Google Sheets: gspread + Google service account
  - Google Drive: Google Drive API for photo backup
  - Hosting: Railway / Fly.io / VPS (~$5/mo)
  - Config: Environment variables (bot token, API keys, sheet ID, admin user IDs)

  What the Bot Does NOT Do

  - No /settle command — results managed manually in sheet
  - No time/match validation (live bets allowed)
  - No currency conversion
  - No cashouts
  - No system bets
  - No balance limits (negative balances allowed)
  - No user self-registration (admin adds manually)
  - No group messages for flagged bets
  - Does not write to STATUS or CASTIG columns
